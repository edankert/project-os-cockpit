# Rename docs-server → project-os-cockpit

## Summary
The project's working name shifts from `docs-server` to `project-os-cockpit`. The original name described what shipped first (a docs render server), but with the cockpit shell, SNAPSHOT-driven home, Library mode, pin button, and SSE live reload all merged, the tool is now the project-os operator UI — its centre pane happens to render Markdown. The new name reflects that. The rename also makes the downstream-pilot deploy path read sensibly: `your-applications.com/tools/project-os-cockpit/` instead of `tools/docs-server/`. `pyproject.toml`, the package directory `src/project_os_cockpit/`, and the entry-point script `project-os-cockpit` had already been migrated in prior work; this change finishes the rename across the repo directory, source-code identifiers, and active prose.

## Impact

### Repo + package
- The repo directory was renamed from `~/Dev/repos/docs-server/` to `~/Dev/repos/project-os-cockpit/`.
- `pyproject.toml` already declared `name = "project-os-cockpit"`, package `project_os_cockpit*`, script `project-os-cockpit = project_os_cockpit.__main__:main`, and the GitHub URL `edankert/project-os-cockpit`. No change here, but the egg-info (`src/project_os_cockpit.egg-info/`) was regenerated by `pip install -e .` to refresh the console-script registration.

### CLI surface (user-visible breaking change)
- `argparse` `prog` flipped from `"docs-server"` to `"project-os-cockpit"`. `--help` output and `--version` (`%(prog)s 0.1.0`) both reflect the new name.
- Console script entry-point is `project-os-cockpit` (was previously `docs-server` in installed environments — anyone who had it installed from a pre-rename checkout has a stale `docs-server` script and should reinstall).
- HTTP `Server:` response header changed from `docs-server/0.1` to `project-os-cockpit/0.1`. Anything keying off the User-Agent-equivalent will need updating.
- Startup log line and the stdout banner now read `project-os-cockpit listening on …` and `project-os-cockpit: http://…/`.
- Shutdown banner now reads `project-os-cockpit: shutting down.`.

### Browser surface (user-visible breaking change)
- `localStorage` theme key changed from `docs-server.theme` → `project-os-cockpit.theme`. **First load after the rename will appear in the user's OS-default theme** (the old key is silently abandoned). Users can re-toggle once and the new key persists. The pinned-paths key (`docs-server.cockpit.pinned-paths`) was similarly renamed to `project-os-cockpit.cockpit.pinned-paths` — pinned-note state will also reset on first load.
- `<meta name="docs-server:source" …>` (consumed by `sse-reload.js`) renamed to `<meta name="project-os-cockpit:source" …>`. The companion JS reader was updated in lockstep.
- Page `<title>` suffix changed from `… — docs-server` to `… — project-os-cockpit`.
- Footer brand-mark now reads `project-os-cockpit`.

### Source-code identifiers
- `templates._PROJECT_NAME` default flipped to `"project-os-cockpit"` (still overridden at server start by `set_project_name(<repo-name>)`).
- `server.DocsServer` class kept under that name for now — it's a legacy identifier consumed by the CLI entry point only and an internal rename can come later without user impact. Module docstrings updated.
- All references to `src/docs_server/` in code-internal comments swept to `src/project_os_cockpit/`.

### Documentation
- `README.md` H1 + invocation examples (`python -m project_os_cockpit …`).
- `CLAUDE.md` project header, summary, and stack notes (`src/project_os_cockpit/`, `python -m project_os_cockpit`).
- `SECURITY.md` prose mentions of the tool.
- `ROADMAP.md` deploy path: `~/Dev/repos/your-applications.com/tools/project-os-cockpit/`.
- `docs/ARCHITECTURE.md`, `docs/GLOSSARY.md`, `docs/PHASES.md` — active reference text updated.
- Active downstream-pilot notes (`FEAT-0005`, `REQ-0017`, `TASK-0020`) and the `PHASE-003-Downstream-Pilot.md` phase note updated to use `tools/project-os-cockpit/` as the shim path.

### Untouched (deliberately)
- `docs/changes/CHG-*` — append-only history. The 12 prior CHG notes still describe their work as "docs-server"; that is correct for the historical record.
- `docs/decisions/ADR-*` — immutable decision records, including `ADR-0001-Custom-Python-Vs-Static-SSG.md` which evaluated "doc-server" as one option. The decision was made under that name.
- `SNAPSHOT.yaml` `items.changes.CHG-20260507-Render-Pipeline-Online.title` and `…-Bootstrap-Python-Project.title` — historical milestone titles preserved verbatim.
